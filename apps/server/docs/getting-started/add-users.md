# Přidání uživatelů

Účastníci jsou klíčovou součástí každé události. Tento návod vám ukáže, jak přidat účastníky a aktivovat jejich účty.

## Typy uživatelů

### Administrátoři
- Mají plný přístup ke správě události
- Mohou přidávat a upravovat účastníky
- Spravují sudy a nastavení
- Vidí všechny statistiky a data

### Účastníci
- Mohou se účastnit událostí
- Jsou sledováni v žebříčku
- Mají omezený přístup k funkcím
- Potřebují registrační token pro aktivaci

## Jak přidat účastníka

### 1. Přístup k přidání účastníka
- Přihlaste se jako administrátor
- Přejděte do sekce **"Účastníci"**
- Klikněte na tlačítko **"Přidat účastníka"**

### 2. Vyplnění údajů
Při přidávání účastníka budete potřebovat:

**Povinné údaje:**
- **Jméno** - Celé jméno účastníka
- **Email** - Emailová adresa (pro registrační token)
- **Pohlaví** - Pro správné statistiky

**Volitelné údaje:**
- **Přezdívka** - Alternativní jméno
- **Poznámka** - Dodatečné informace

### 3. Generování registračního tokenu
Po přidání účastníka:
- Systém automaticky vygeneruje registrační token
- Token se odešle na email účastníka
- Účastník musí token použít pro aktivaci účtu

## Aktivace účtu účastníka

### Pro účastníka:
1. **Přijetí emailu** - Účastník obdrží email s registračním tokenem
2. **Kliknutí na odkaz** - V emailu je odkaz pro aktivaci
3. **Nastavení hesla** - Účastník si nastaví své heslo
4. **Přihlášení** - Účet je aktivní a účastník se může přihlásit

### Pro administrátora:
- Můžete sledovat stav aktivace účtů
- Vidíte, kteří účastníci ještě neaktivovali své účty
- Můžete znovu odeslat registrační token

## Správa účastníků

### Základní operace:
- **Editace** - Změna údajů účastníka
- **Deaktivace** - Dočasné pozastavení účtu
- **Smazání** - Trvalé odstranění účtu
- **Reset hesla** - Vynucení změny hesla

### Hromadné operace:
- **Import účastníků** - Nahrání ze souboru
- **Export seznamu** - Stažení seznamu účastníků
- **Hromadné odeslání tokenů** - Odeslání tokenů všem neaktivovaným

## Stav účtu

Účet účastníka může být v různých stavech:

- **Neaktivní** - Účet je vytvořen, ale není aktivován
- **Aktivní** - Účet je aktivní a účastník se může přihlásit
- **Deaktivovaný** - Účet je dočasně pozastaven
- **Blokovaný** - Účet je trvale zablokován

## Bezpečnost

### Registrační tokeny:
- Tokeny mají omezenou platnost (obvykle 24 hodin)
- Každý token je unikátní a nelze ho použít vícekrát
- Tokeny se automaticky invalidují po použití

### Hesla:
- Účastníci si nastavují vlastní hesla
- Hesla musí splňovat bezpečnostní požadavky
- Administrátoři nemohou vidět hesla účastníků

## Co dál?

2. [Správa sudů](../user-guide/barrels.md) - Přidejte sudy pro konzumaci
3. [Žebříček](../user-guide/leaderboard.md) - Sledujte statistiky

## Tipy

- **Email** - Ujistěte se, že emailová adresa je správná
- **Tokeny** - Sledujte, kteří účastníci ještě neaktivovali účty
- **Hromadné operace** - Pro větší události použijte import ze souboru
- **Komunikace** - Informujte účastníky o nutnosti aktivace účtu
